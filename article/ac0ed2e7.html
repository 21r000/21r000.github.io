# 概述

PE（Portable Execute）文件是 Windows 下可移植、可执行文件的总称，常见的有 DLL，EXE，OCX，SYS，COM 等都是 PE 文件，PE 文件是微软 Windows 操作系统上的程序文件（可能是间接被执行，如 DLL）。

而事实上，一个文件是否是 PE 文件与其扩展名无关，PE 文件可以是任何扩展名。

PE 文件的结构一般来说如下图所示：从起始位置开始依次是 MS-DOS 头，标准 PE 头，扩展 PE 头，数据目录，以及节表。

# MS-DOS

使用 010 打开一个 exe 文件可以看到 MS-DOS 头

MS-DOS 头出于兼容性的目的而存在，主要是为了防止 PE 文件运行在 MS-DOS 系统下出现错误。

# PE 头结构

PE 头分为：

  • 标准 PE 头：每一个 PE 文件中都有。
  • 扩展 PE 头：可有可无。

_IMAGE_NT_HEADERS ，分为 32 位和 64 位两个版本，根据逆向解析的程序使用相应的版本。

PE 标识：5045

typedef struct _IMAGE_OPTIONAL_HEADER 定义

AddressOfEntryPoint：程序入口，当前 PE 文件从哪个位置开始执行，配合 ImageBase (内存镜像基址) 来看

  • VA：Virtual Adress 00000000h-0fffffffh 进程的基地址 + 相对虚拟内存地址
  • RVA ：Reversc Virtual Adress 相对虚拟内存地址 针对某一个模块而存在
  • FOA：File Offset Address 文件偏移地址
成员大小 (十进制)说明
Magic2 字节PE32：10B、PE32+(64)：20B，可以用来确认是 32 位还是 64 位程序
MajorLinkerVersion1 字节链接器版本号
MinorLinkerVersion1 字节链接器版本号
SizeOfCode4 字节所有代码节的总和，文件对齐后的大小，由编译器填写
SizeOfInitializedData4 字节包含所有已经初始化数据的节的总大小，文件对齐后的大小，由编译器填写
