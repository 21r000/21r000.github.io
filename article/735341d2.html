# 资源表结构解析

Lord PE 中查看资源表

资源在 PE 中是以目录的形式存在的，一般有 3 层：

  • 资源类型
  • 目标资源 ID
  • 资源代码页

资源表头部结构： PIMAGE_RESOURCE_DIRECTORY 负责指出后面数组中的成员个数

  • Characteristics：资源属性，一般情况下为 00000000
  • TimeDateStamp：资源创建的时间
  • MajorVersion：主版本号，一般情况下为 4
  • MinorVersion：子版本号，一般情况下为 0
  • NumberOfNamedEntries：以字符串作为资源标识的条目的个数
  • NumberOfIdEntries：以数字作为资源标识的条目的个数

结构数组： PIMAGE_RESOURCE_DIRECTORY_ENTRY 数组成员分别指向下一层目录结构

PIMAGE_RESOURCE_DIR_STRING_U

资源数据： PIMAGE_RESOURCE_DATA_ENTRY

# 报错

出现这个问题的原因是：

从 VS2017 起，编译器要求通过使用字符串文本初始化的指针符合严格的 const 限定。所以会自动添加 /Zc:strictStrings 选项，你可能在上面没找到 /Zc:strictStrings ，其实它是通过 /permissive- 来隐式设置此选项。

# 解决方法

打开项目属性

在配置属性中打开 C/C++，找到命令行后在它的 "其它选项" 中添加 /Zc:strictStrings- 即可
