通过这篇文章我们来了解一下如何更为简便的调用我们 windows 的一个 API。

在之前的学习中我们就已经知道了如何使用 API，但是要求我们先包含各种各样的东西，再接着写的那种方式操作起来并不容易。比如说你要先声明，然后在声明完之后，你要把类型都标记出来，然后再写。

那么有没有一种方法就是直接可以调用 win32 的 API 的呢，其实呢这种方法也是有的，那么接下来我们一起来学习一下如和配置这种环境。

首先呢，我们还是正常的新建一个 asm 的一个项目，具体流程参考这篇文章中的汇编语言环境搭建

# 安装 masm32

创建完成之后，我们想要更简便的使用 win32 的 api，我们还要借助 masm 的变种 masm32

点击 Download

下载我们所需要的版本

下载完成之后，双击打开得到一个 install 文件

双击进行安装

点击 install，选择安装目录

注：目录要自己记得下面还要用到！

一路默认确定即可

注：若安装时间较长，请耐心等候。

安装完成显示

安装完成之后会在你所选择的目录下生成一个 masm32 文件夹

点进去可以看到各种配置文件

inculde 和 lib 目录是我们接下来会用到的目录

# 配置所需 Include Path 和附加库

接下来打开我们的项目右键打开属性面板

在 Microsoft Macro Assembler 中选择 General ，并配置它的 Include Path 变量

点击 <编辑...>

进行如下勾选，一般为默认

然后我们将 masm32 的 include 文件夹路径填写进去之后，显示如下

点击确定之后显示如下

点击 “应用”

接下来我们打开 “链接器” 的 “常规” 面板配置其 “附加库目录”

进入编辑模式后点击右上角的 “..."，选择所需文件夹

找到 masm32 的 lib 目录

结果显示

依次点击 "应用"->"确定" 即可。

如此我们就能更加方便的使用 win32 的 api 了。

# 关于报错

错误原因
高于 14.26.28801 的 msvc 工具集编译不了 masm32v11r 环境的汇编

解决方案

1. 更改 msvc 工具集版本
打开：Visual Studio Installer，选择修改输入 MSVC，安装 MSVC v14x - VS 201x C++ x64/x86 生成工具(v14.xxxx) 以下任意一个版本的工具即可：

等待修改完成即可

2. 修改项目的 MSVC 的版本
右键项目 -> 属性 -> 配置属性 -> 高级 ->MSVC 工具集版本 -> 选择低于或等于 14.26 的 msvc 工具集即可。

注：强烈建议使用2015或2017版本，不存在高配置不兼容问题！

（ps：出现以上问题不排除可能因为小编实力没到位的问题 QAQ！！！）

这里使用 2015 版的配置一下就好了，没有什么奇奇怪怪的问题，还可以安装 Asm 高亮渲染插件！
