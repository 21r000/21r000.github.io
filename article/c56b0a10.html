# 寻找 main 函数

对 Debug 和 Release 进行初步了解

  • Debug ：带调试信息，不会进行太多的优化

  • Release ：对程序进行优化，精简代码，合并或取消重复的、无意义的函数，使生成的程序更小，运行速度更快

在分析程序时一般为 Release 版本的程序但同时也不排除有 Debug 版本的程序，在程序分析过程中我们要先找到我们所需要的代码也即 主函数main 中的代码。

准备两个版本的运行程序：

  • D.exe ： Debug 版本的运行程序
  • R.exe ： Release 版本的运行程序

# Ollydbg 打开

通过 Ollydbg 打开 D.exe 程序来寻找它的 主函数main

# 第一种方式

在 Ollydbg 中右键 "查找"->"所有参考文本字符串"

可以看到我们想要的字符串

双击即可进入 main函数

# 第二种方式

打开后在第一行的 call 位置按 " F8 " 进行跳转

再次按 " F8 "，可以发现右侧注释部分出现红色

多次点击 " F8 " 到红色注释部分后

按 " F7 "进入后一路" F8 "

发现右侧注释部分再次出现红色注释

再次点击 " F8 "到该处，若能跳转到此处则按" F7 "进入，如若不然则继续" F8 " 直到能跳转

按 " F7 " 进入后

继续一路 " F8 "，可以发现三个" push "，一个" call "，我们知道完整的 main函数 是 存在三个完整的参数 的

此时我们可以标记一下： 此处疑似main函数

此时在此处按 " F7 " 进入

可以发现 " 21r000 " 字样，而旁边的红色注释则是条用的 main函数

同时 Release 版本还可以在打开的页面直接进行查找

# ida 打开

打开 ida 后直接将文件拖入后

自动找到 main 函数

此外 ida 的 " F5 " 也可带你进入 main函数 。
